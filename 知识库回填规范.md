# 知识库回填规范

## 目标

把本学期课件、PPT、讲义中的内容整理成适合长期复习的 Obsidian 知识库。

目标不是复制课件，而是建立：

- 章节笔记
- 知识点卡片
- 题型卡片
- 易错点总表
- 课程首页
- 复习索引
- 考前急救入口

## 处理顺序

推荐按课程分批处理：

1. 数学分析
2. 电路基础
3. 大学物理
4. 概率统计
5. C++ 与数据结构
6. 通识类课程

每次只处理一门课，确认质量后再处理下一门课。

## 每门课处理流程

1. 扫描该课程课件、PPT、讲义。
2. 更新 [[课件处理进度表|课件处理进度表]]。
3. 建立或更新课程首页。
4. 建立或更新章节笔记。
5. 从课件中提炼知识点卡片。
6. 从课件例题中总结题型卡片。
7. 更新 我的易错点总表。
8. 更新 [[题型卡片总索引|题型卡片总索引]]。
9. 更新 [[知识点卡片总索引|知识点卡片总索引]]。
10. 更新 [[考前急救|考前急救]]。
11. 更新 [[我总是搞混的东西|我总是搞混的东西]]。
12. 更新 [[待补全清单|待补全清单]]。
13. 检查双链、重复文件、公式格式。
14. GitHub 同步。
15. 网站 build 和 push。

## 公开网站内容安全规则

不要放入公开网站：

- 课件原文大段复制
- 教材扫描内容
- 作业截图
- 老师姓名
- 班级群信息
- 学号
- 电话号码
- 地址
- 非公开资料原图
- 版权风险较高的大段内容

可以公开：

- 自己整理的知识点
- 自己总结的题型方法
- 自己归纳的易错点
- 自己改写过的例题思路
- 通用公式、定理、方法框架

## 例题处理规则

从 PPT 中选例题时，不要大段复制原题。应整理为：

- 题目简述
- 解题思路
- 关键步骤
- 结论
- 易错点

如果 PPT 中没有对应例题，不要自己编造具体例题。可以先创建题型卡片框架，并标注“待补充例题”。

## 重复文件处理规则

如果发现同一知识点或题型存在多个相似文件，优先合并到命名最规范的文件中。

合并原则：

1. 保留已有有效内容。
2. 合并重复内容。
3. 删除或重定向重复文件。
4. 在最终总结中说明处理情况。

## 待人工确认标记

遇到不确定内容时，不要硬写死。

统一使用：

> [!warning] 待人工确认
> 这里需要人工核对。

或：

```markdown
%% TODO: 需要人工核对 %%
```

## 质量自检要求

每次处理完一门课后，必须输出质量自检报告：

1. 哪些内容是根据课件明确整理出来的？
2. 哪些内容是根据通用课程知识补充的？
3. 哪些题型卡片有 PPT 例题支撑？
4. 哪些题型卡片只是先建了框架，还缺例题？
5. 哪些文件可能存在重复或需要人工确认？
6. 有没有大段复制课件原文？
7. 有没有公式格式不符合 Obsidian 规范？
8. 有没有死链或疑似错误双链？
9. 哪些页面最值得优先人工检查？
